The constraints

What made this interesting.

The usual options were off the table, which shaped what a good answer looked like.

$

No budget

No software procurement, no contractors. The fix had to come from within the team, using tools already on hand.

⊘

No cloud, no network

No syncing, no analytics, no remote fonts. The tool needed to work entirely offline.

⌘

No install rights

CSR workstations were locked down. Whatever I built had to run from a regular file in the default browser.

⎋

No saved data

Privacy review meant no storage of any kind — no localStorage, no filesystem, not even autofill. Each note had to exist for one call and disappear.

How it works

Privacy built into the structure.

The tool is a single HTML file with no dependencies and no build step. Fields are rebuilt on every page load, autofill is disabled at the attribute level, and there's no persistence layer to begin with.

Because nothing is stored, there's nothing to leak.

// Disable autofill and persistence on every input. document.querySelectorAll('input, textarea') .forEach(el => { el.setAttribute('autocomplete', 'off'); el.setAttribute('data-lpignore', 'true'); el.setAttribute('spellcheck', 'false'); }); // Block network calls at the primitive level. if (typeof fetch !== 'undefined') { window.fetch = () => { throw new Error('Network disabled'); }; }
